# Casa della Salute

A web-based management system for a health centre (Casa della Salute), handling medical visits, nursing services, bookings, and patient records. Built with Django following the Model-View-Template (MVT) architecture.

## Overview

The system is designed to be deployed on terminals within the health centre and accessed by both patients and staff. Credentials are generated by the secretariat and distributed to users.

The facility includes 3 adult consultation rooms, 2 rooms for blood draws and dressings, and 1 paediatric consultation room.

---

## User Roles

**Patient** — can log in to view personal data, upcoming and past visits, active bookings, and book new visits or services. Email confirmation is sent upon booking. Paediatric patients have a linked adult guardian.

**Doctor** — can view their schedule, associated patients, and planned absences.

**Nurse** — can view personal data and the list of services they have performed.

**Secretariat** — admin role with full management access: registering and editing users, inserting visit outcomes, managing bookings, scheduling nurse shifts, and recording doctor absences.

## Tech Stack

| Component | Technology |
|---|---|
| Framework | Django (MVT pattern) |
| Language | Python |
| Database | Django ORM (relational) |
| Environment management | python-dotenv |
| Version control | Git / GitHub |

### Design Patterns Used
- **MVT (Model-View-Template)** — Django's native architecture
- **Singleton** — centralised configuration management
- **Observer** — Django signals (`post_save`, `pre_delete`) for automated actions (e.g. booking notifications)
- **Adapter** — integration of external libraries for form validation and UI

## Database Structure

The data model is centred on a `Utente` (User) entity, from which all user types inherit:

```
Utente
├── Paziente (Patient)
├── Medico (Doctor)
├── Infermiere (Nurse)
└── Segreteria (Secretariat)
```

### Key Entities
- `Utente` — username, nome, cognome, codiceFiscale
- `Paziente` — dataNascita, luogoNascita, email, referenteAdulto, medicoConvenzionato
- `Medico` — specialità, assenzePianificate
- `Infermiere` — giorniServizio
- `Segreteria` — ruolo
- `Prenotazione` (Booking) — id, data, ora, tipo, richiedente
- `Visita` (Visit) — id, esito, idPrenotazione, medico, paziente, ambulatorio
- `Prestazione` (Service) — id, data, esito, note, idPrenotazione, infermiere, paziente, ambulatorio
- `Ambulatorio` (Room) — id, tipo
- `Sostituzione` (Doctor substitution) — medicoSostituto, medicoSostituito

## Setup & Installation

### Prerequisites
- Python 3.x
- pip

### Install dependencies

```bash
pip install -r requirements.txt
```

### Database setup

```bash
python manage.py makemigrations
python manage.py migrate
```

### Create a superuser (Secretariat admin)

```bash
python manage.py createsuperuser
```

### Run the development server

```bash
python manage.py runserver
```

The system will be available at: http://127.0.0.1:8000

---

## Login

Select the correct role (patient, doctor, nurse, or secretariat) at login. Credentials for patients are generated by the secretariat; staff credentials are set up during system configuration.
